# 海底神殿の秘宝 — 実装計画

## プロジェクト概要

「海底神殿の秘宝」はデッキ構築型カードゲームのブラウザ実装です。
単一 HTML ファイル（`index.html`）として完結させ、外部ライブラリ不要で動作します。
プロトタイプ（`prototype/海底神殿の秘宝.html`）を土台に、仕様書（`doc/海底神殿の秘宝.md`）に沿って段階的に完成させます。

## 実装ステップ

### Step 1 — データ定義・ゲームエンジン基盤

**目標**: ゲームの中核となるデータ構造とロジックを JavaScript で定義する。

- [ ] 1-1. カードデータ定義（全カード種別・パラメータ）
  - スターターカード 4 種（初級探索者 ×6、古びた旧神の印 ×2、ナイフ ×2、初級防具 ×2）
  - 港カード 8 種（旧神の印、バックパック、手斧、大剣、中級防具、重装甲、小火器、重火器）
  - 探索者カード 5 種（精神科医、中級、格闘、火器、重火器）
  - モンスターカード 13 種（強さ 6〜25）
  - 宝物カード 3 種（宝冠、宝盃、宝剣）
  - 秘宝カード 2 種（銀の鍵、輝くトラペゾヘドロン）
  - 狂気カード（パラメータなし・捨て札専用）

- [ ] 1-2. ゲーム状態オブジェクト定義
  - プレイヤー情報（名前、アイコン、種別 human/AI、デッキ、手札、捨て札、除外済み、獲得カード）
  - 海底神殿盤面（モンスターデッキ、秘宝デッキ、現在オープン中のカード）
  - 港盤面（探索者山札 4 種、港カード山札 8 種、スターターカード山札）
  - ターン管理（現在プレイヤーインデックス、フェーズ）

- [ ] 1-3. デッキ操作ユーティリティ
  - シャッフル（Fisher-Yates）
  - ドロー（山札枯渇時に捨て札をシャッフルして補充）
  - 捨て札へ送る、除外する

- [ ] 1-4. 初期化処理
  - プレイヤー人数・設定に基づいてデッキ枚数を計算
  - モンスターデッキ（強さ 15 以下 + 宝物 3 種）と秘宝デッキ（強さ 25 × 3 + 秘宝 2 種）を構築
  - 最初のカードを伏せて盤面に配置
  - 各プレイヤーのスターターデッキを作り、8 枚ドロー

### Step 2 — タイトル画面・セットアップ画面

**目標**: ゲーム開始前の設定 UI を完成させる。

- [ ] 2-1. タイトル画面レイアウト

  - ゲームタイトル（「海底神殿の秘宝」）
  - サブタイトル（「Ocean Monument — Hidden Treasure」）
  - フレーバーテキスト（ストーリー紹介）

- [ ] 2-2. タイトルボタン
  - 「ゲーム開始」ボタン → セットアップ画面へ
  - 「図鑑」ボタン → 図鑑画面へ
  - 「遊び方」ボタン → 説明書モーダルへ

- [ ] 2-3. セットアップ画面（プレイヤー設定）
  - プレイ人数選択（2〜5 人）
  - 各プレイヤーの設定スロット
    - 人間 / AI 切り替え
    - AI 強さ選択（ノーマル / ハード / ストロング）
    - アイコン選択（🐋🐬🦭🐟🐠🐡🦈🐙🐚🪸🪼🦀🦞🦐🦑🦪 から選択）
    - プレイヤー名入力
  - プレイ順（ランダム / 指定）の選択
  - 「ゲーム開始」ボタン → ゲーム初期化 → ゲーム画面へ

### Step 3 — 図鑑画面

**目標**: モンスター・宝物・秘宝の画像付き一覧を実装する。

- [ ] 3-1. 図鑑一覧表示

### Step 4 — ゲーム画面レイアウト

**目標**: ゲーム画面の骨格 UI を構築する。

- [ ] 4-1. 画面全体グリッド
  - 上部：ターン情報バー（現在プレイヤー・フェーズ表示）
  - 上部：行動選択ボタン（港を訪れる / 海底神殿に入る / 休息する）
  - 中央左：海底神殿盤面・港盤面
  - 中央右：行動ログパネル（全プレイヤーの行動履歴）
  - 下部：手札・装備表示エリア

- [ ] 4-2. ターン情報バー
  - 現在のプレイヤーアイコン・名前
  - フェーズ名（行動選択中 / 港 / 海底神殿 / 休息 / AIターン）
  - ターン数・ラウンド数

- [ ] 4-3. 行動選択ボタンエリア（画面上部固定）
  - 各ボタンにカーソルを当てると行動の説明をツールチップ表示
  - 選択中の行動をハイライト
  - 他プレイヤーのターン中は非表示 or グレーアウト

- [ ] 4-4. 海底神殿盤面
  - 現在の神殿カード（伏せ/オープン表示）
  - カード残数表示（モンスターデッキ / 秘宝デッキ）

- [ ] 4-5. 港盤面
  - 探索者山札 × 4 種（一番上を表示、残枚数表示）
  - 港カード山札 × 8 種（一番上を表示、残枚数表示）
  - スターターカード × 4 種

- [ ] 4-6. プレイヤーデッキ確認ボタン
  - 「全カード」「山札（名前順）」「捨て札」を別タブで参照できるモーダル

- [ ] 4-7. 手札エリア
  - 8 枚の手札をカードとして表示
  - マウスオーバーでカード詳細（効果、相性情報）を表示
  - 他プレイヤーの手札は参照不可（伏せ表示）

### Step 5 — 港アクション

**目標**: 「港を訪れる」の全処理を実装する。

- [ ] 5-1. 手札公開・港効果の適用
  - 手札を全公開
  - 港効果（金貨生成・カードドロー等）を選択的に適用できる UI
  - 金貨累積表示

- [ ] 5-2. 合計金貨値の計算
  - 場に残っているカードの金貨値を合計

- [ ] 5-3. カード購入 UI
  - 購入可能なカードのみ選択可能（コスト ≤ 合計金貨値）
  - 購入したカードは捨て山へ
  - 購入後、残余金貨は消滅

- [ ] 5-4. 精神科医による狂気除去
  - 精神科医が手札にある場合、狂気カードの除去選択肢を表示

- [ ] 5-5. ターン終了処理
  - 全手札を捨て山へ
  - 8 枚ドロー

---

### Step 6 — 海底神殿アクション

**目標**: 「海底神殿に入る」の全処理を実装する。

- [ ] 6-1. 手札公開・神殿効果の適用
  - 手札を全公開
  - 海底神殿効果の適用

- [ ] 6-2. 装備割り当て UI
  - 探索者一覧を表示（全員で神殿に挑む）
  - 各探索者に武器 / 防具 / 道具をドラッグ or ボタン選択で装備
  - 体力 ≥ 重さ のチェック
  - 火器は「火器使用可能」探索者のみ装備可
  - 各装備は 1 人 1 つまで（共有不可）

- [ ] 6-3. モンスターカードをオープン
  - 伏せカードを公開
  - 宝物・秘宝の場合は即座に獲得（演出あり）
  - モンスターの場合は戦闘フェーズへ

- [ ] 6-4. 戦闘処理
  - 「カードを引く」効果を先に解決（ドロー後に戦闘力再計算）
  - 明かりペナルティ計算（階層別 -2/-4/-6、明り 1 毎に +2 緩和）
  - 合計戦闘力 vs モンスター強さ
  - 勝利：モンスターカード + 狂気 1 枚を捨て札へ。新カードを伏せて配置
  - 敗北：モンスターカードをオープンのまま維持。狂気 2 枚を捨て札へ

- [ ] 6-5. 戦闘演出
  - モンスターと探索者のカード表示
  - 戦闘力計算アニメーション
  - 勝利 / 敗北演出
  - 狂気付与の演出
  - 宝物獲得時：暗い祝福演出
  - 秘宝獲得時：クリア演出 → ゲーム終了

- [ ] 6-6. ターン終了処理
  - 全手札を捨て山へ
  - 8 枚ドロー

---

### Step 7 — 休息アクション

**目標**: 「休息する」の全処理を実装する。

- [ ] 7-1. 破棄 UI（無料・最大 2 枚）
  - 狂気 2 枚 or 狂気以外 2 枚のみ（混合不可）
  - 選択したカードを除外（デッキから永久に消える）

- [ ] 7-2. 強化 UI（金貨値 10 消費・何枚でも可）
  - 手札から探索者を選択
  - 金貨値 10 を消費して効果 +1 強化（例：攻撃 1 → 攻撃 2）
  - 残余金貨が 10 以上ある限り続けて強化可能

- [ ] 7-3. ターン終了処理
  - 全手札を捨て山へ
  - 8 枚ドロー

### Step 8 — AI プレイヤー

**目標**: 3 段階の難易度を持つ AI を実装する。

- [ ] 8-1. AI 共通ロジック
  - 手番時に行動選択（港 / 海底神殿 / 休息）
  - 各アクションの自動処理

- [ ] 8-2. ノーマル AI
  - 基本的に金貨が溜まれば購入、戦闘力が足りれば挑む
  - ランダム要素を多めに

- [ ] 8-3. ハード AI
  - デッキ圧縮（不要カードを破棄）を優先
  - 戦闘力計算をより精密に

- [ ] 8-4. ストロング AI
  - 最適な装備割り当て
  - 勝利点効率を考慮した購入判断
  - 「カードを引く」効果を最大活用

- [ ] 8-5. AI ターン演出
  - 他プレイヤーのターン中であることが分かる表示
  - AI の行動内容をログに表示

### Step 9 — ゲーム終了・結果画面

**目標**: ゲーム終了条件の判定と結果表示を実装する。

- [ ] 9-1. 終了条件
  - 秘宝（銀の鍵 or 輝くトラペゾヘドロン）を誰かが獲得した瞬間にゲーム終了

- [ ] 9-2. 勝利点集計

    - 宝物カードの勝利点
    - 秘宝カードの勝利点（輝くトラペゾヘドロンは狂気 1 枚 = +1 点）

- [ ] 9-3. 結果画面
  - 全プレイヤーの順位・勝利点・獲得カード一覧を表示
  - 最高得点者をハイライト
  - 「最終盤面を見る」ボタン（ゲーム画面に戻って閲覧可能）
  - 「タイトルへ戻る」ボタン

## 技術方針

| 項目 | 方針 |
| 実装形式 | 単一 HTML ファイル（CSS・JS インライン） |
| 外部依存 | Google Fonts のみ（CDN）|
| フレームワーク | バニラ JS（フレームワーク不使用） |
| 画像 | `image/` ディレクトリの既存 JPG を使用 |
| サウンド | Web Audio API で効果音を生成 |
| 対応ブラウザ | Chrome / Edge 最新版 |
